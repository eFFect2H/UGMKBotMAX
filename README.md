# UGMKBotMAX (Max.Bot.SDK)

Проект-шаблон под ТЗ для бота **«Заявки УГМК-Здоровье»** на основе `Max.Bot.SDK`.

## Что реализовано

- Диалоговая модель заявки (объект → направление → описание → фото → контакты).
- Автомаршрутизация по чатам из конфигурации.
- Логический слой для записи в Google Sheets.
- Админ-функции:
  - закрытие заявки по ID;
  - формирование списка открытых заявок;
  - ежедневные напоминания по открытым заявкам;
  - пометка просроченных заявок.
- Заготовка еженедельной выгрузки в Excel/PDF (в текущем виде: CSV-bytes и text-bytes).

> Внимание: `MaxBotGateway` и `GoogleSheetsService` содержат интеграционные заглушки с точками расширения для боевого подключения SDK/API.

## Структура

- `src/UGMKBotMAX/Domain` — доменные сущности заявок.
- `src/UGMKBotMAX/Application` — оркестрация сценариев, интерфейсы и фоновые workers.
- `src/UGMKBotMAX/Infrastructure` — in-memory репозиторий, роутер, gateway к Max.Bot.
- `src/UGMKBotMAX/Integration` — интеграции Google Sheets и отчетов.
- `src/UGMKBotMAX/Config` — strongly-typed настройки.

## Запуск

1. Заполнить `appsettings.json` (токен бота, `SpreadsheetId`, путь до Service Account JSON).
2. Подключить реальные обработчики Max.Bot.SDK в `MaxBotGateway`.
3. Подключить реальный append в `GoogleSheetsService`.
4. Запустить worker service.

## Команды (целевая логика)

- `/start` — приветствие.
- `/new` — создать заявку.
- `/close <guid>` — закрыть заявку.
- `/report` или `/отчет` — список открытых заявок.
